GUIDENCE TO USE PYBOTMAKER

PyBotMaker shows you how to build a ChatBot. You'll learn How to create Agent, Intent,Training Phrases,Responses,Action and Parameters , Entity through the PyBotMaker.

AGENT

Agents are the modules ie., your project name.

These modules can be included in your app, website, product, or service and translate text or spoken user requests into actionable data. This translation occurs when a user's utterance matches an intent within your agent. The matched intent then delivers a response back to the user. This response can be a simple text or spoken acknowledgment or a webhook response that includes information obtained from another system.

CREATE AN AGENT

To create an agent, follow these steps:

click on create Agent.

Enter a name for your agent and Click CREATE.

INTENT

An Intent is a specific action that the user can invoke by using one of the defined terms.

Intents in your agent that map user input to responses. In each intent, you define examples of training phrases, action and parameters, responses that can trigger the intent, what to extract from the utterance, and how to respond.

For example, the user could ask “I want to register now”.if these are terms defined within the intent training phrases, then they will be detected and that defined intent under will be triggered.

Intents would be your project sub modules for example registration and login

CREATE AN INTENT

To create an Intent, follow these steps:

Click Create INTENT

Enter a name for your intent.

On the left menu click the Respective intent.

TRAINING PHRASES

Training phrases are collections of possible utterances.

Training phrases are collections of possible utterances that users might say to match an intent. 

This allow us to define different phrases which can be said by the user in order to trigger and match our intent

ACTION AND PARAMETERS

Defines how relevant information (parameters) are extracted from user utterances

Required defines whether the parameter is required or not. Parameter Name is the name of the parameter in REST API.Entity can be user defined or system entities, user can pick according to usage.The value of a parameter can be used in responses to refer to that parameter. The value is like a placeholder or the variable assigned to the parameter.Prompts is the utterance that's spoken or displayed back to the user.

RESPONSES

Responses will deliver to our user once the request for fulfilment has completed

In the Responses section of your intent, you can define one or more static text responses that will be returned when a user's input matches that particular intent.  

ENTITY

Entities are used to pick out specific pieces of information that your users mention.

When the user speaks or types their request, It will look for the entity and the value of the given entity can be used within the request.It already contains a set of pre-defined system entities which can be used when constructing intent. If these are not enough, we also have the ability to define our own entities for use within our intents  

CREATING ENTITY

To create an Entity, follow these steps:

Click ENTITIES. 

Enter Your Entity. 

Click CREATE.
